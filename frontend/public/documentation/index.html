Core Platform Features

What is this platform?

A full-stack, enterprise-grade B2B marketplace bridging global manufacturers (Suppliers) and bulk procurement professionals (Buyers), complete with escrow payments, real-time messaging, and a powerful Admin moderation layer.

  • Multi-Role System: Specialized portals and dashboards for Buyers, Suppliers, and Global Administrators.
  • Dynamic Catalog & Product Tiers: Multi-level category taxonomies with tiered pricing arrays, MOQ logic, and product customization queries.
  • RFQ & Negotiation Engine: Real-time messaging and Request for Quotation (RFQ) channels powered by Socket.io for seamless custom price agreements.
  • End-to-End Visual Order Tracking: Milestone-based timeline tracking (Placed → Paid → Processing → Shipped → Delivered).
  • Multi-Gateway Payment Integration: Pre-integrated Stripe, PayPal, and Razorpay checkout flows.
  • Robust Security & Verification: JWT session security, ReCAPTCHA v3 bot prevention, social logins (Facebook, LinkedIn), and built-in Admin KYC verification.
  • Global Localization: Multi-language display, dynamic currency conversions, and automated transactional email delivery via SMTP.
  • Premium Mobile-First Design: Responsive design with dynamic headers, smart search container layout, and mobile-optimized image gallery.

1. Full Project Setup & Installation

Prerequisites & Technology Stack

Before installing, ensure your environment meets the following requirements:

Node.js v18/20 LTS
MongoDB v6.0+
NPM / Yarn
React / Next.js
Nginx / Apache
PM2 Process Manager

Environment Variables (.env Configuration)

Both the frontend and backend require properly configured environments. Example environment variables:

backend/.env
PORT=5000 MONGODB_URI=mongodb://localhost:27017/alibaba_demo JWT_SECRET=your_super_secret_jwt_key STRIPE_SECRET_KEY=sk_test_... PAYPAL_CLIENT_ID=AcM... PAYPAL_APP_SECRET=EOf... FRONTEND_URL=http://localhost:3000
frontend/.env
REACT_APP_API_URL=http://localhost:5000/api REACT_APP_RECAPTCHA_SITE_KEY=your_recaptcha_v3_site_key
Security Warning

Never commit your actual live .env files to Git. Always use .env.example as a template and add .env to your .gitignore.

Backend Installation (Node.js / Express)

Follow these steps to initialize the core REST API backend.

bash
cd backend/ npm install # Run local development server with hot-reload (Nodemon) npm run dev # Start production server npm start

Frontend Installation (React.js / Next.js)

Start the user interface application which powers the Buyer, Supplier, and Admin portals.

bash
cd frontend/ npm install npm start

The marketplace will open at http://localhost:3000.

Database Seeding & Storage Setup

Populate your fresh MongoDB instance with default Admin users, predefined categories, and system settings:

bash
cd backend/ npm run seed

Storage Setup: Product images, invoices, and supplier documents are stored locally in backend/uploads or AWS S3/Cloudinary depending on your configured driver in .env. Ensure the uploads/ folder has public read permissions.

Queue & Cron Job Setup

Cron jobs are initialized automatically upon starting the server, handling background tasks such as:

  • Auto-canceling unpaid pending orders after 24 hours.
  • Compiling weekly analytics & sending summary emails.
  • Re-syncing subscription status for expired supplier plans.

Build & Deployment Steps (Production)

Deploy to a live Ubuntu/Linux server by creating a production build and using PM2 for the backend process manager.

bash
# 1. Build the Frontend cd frontend/ npm run build # Outputs optimized static files to /build # 2. Start Backend with PM2 cd backend/ npm install -g pm2 pm2 start server.js --name "alibaba-backend" pm2 save pm2 startup

Configure Nginx to serve the frontend/build folder on port 80/443, and proxy /api requests to localhost:5000.

Common Troubleshooting Steps

CORS Errors? Ensure your FRONTEND_URL in the backend .env precisely matches the domain the React app is served from (including http vs https).
Images not loading? Verify that your backend URL resolving static assets matches the explicit server IP or domain. Check backend/uploads/ folder permissions.
Socket.io not connecting? Ensure the Nginx config includes WebSocket proxy headers: proxy_set_header Upgrade $http_upgrade;

2. User Flows & System Architecture

End-to-End Transaction Lifecycle

For Clients (Business Logic)

The platform operates as an Escrow-driven marketplace. Buyers find products, place an order, and their payment is held securely. The Supplier ships the item. Only once the Buyer confirms receipt does the platform release funds to the Supplier's virtual wallet, minus the marketplace commission.

For Developers (Technical Architecture)

Cart Splitting: The MongoDB Cart model splits items by supplier_id. Checkout generates one parent Transaction and multiple child Order documents.

Payment Escrow: Stripe/CCAvenue webhooks mark the Order as paid while holding distribution in a pending ledger.

Wallet DB Ledger: When Buyer hits "Confirm Receipt", the backend calculates admin commission from Settings and increments the Supplier's User.walletBalance.

Admin Workflow

For Clients

Administrators are the overseers. Admins verify new suppliers to prevent fraud, moderate product listings, resolve buyer-supplier disputes, and manage website content (banners, global taxes, categories).

For Developers

RBAC: Admins have JWT payload role: 'admin'. The backend isAdmin middleware protects all /api/admin/* HTTP routes.

Dynamic Configuration: Settings like global tax, categories, and hero sliders are fetched from MongoDB Settings collections and mutated via the React dashboard.

Buyer Workflow

For Clients

Buyers log in to source products across global verified suppliers. They can add ready-to-ship products to their cart, or for custom bulk orders, submit a Request for Quotation (RFQ) directly to a supplier to negotiate pricing.

For Developers

RFQ Tracking: Bypasses standard cart logic. Mongoose generates a Quotation document. Communication runs through Socket.io until an agreed custom invoice is generated.

AI Search Layer: Leverages a keyword heuristic route to map conversational queries into exact Product schema $regex matches.

Supplier Workflow

For Clients

Suppliers operate an independent storefront within the marketplace. They undergo KYC verification, upload catalogs with bulk-pricing tiers, fulfill orders with tracking numbers, and withdraw earnings natively via the dashboard.

For Developers

Schema Ownership: Product models require owner: ObjectId referencing the Supplier. A Supplier can only mutate their own products.

Withdrawal System: Funds exist as virtual integers in the DB until a Supplier submits a WithdrawalRequest. Admin manually approves off-platform.

6. Frequently Asked Questions

FAQ

Q: How do I change the marketplace commission? Navigate to Admin › Settings › General Config. Update the "Commission Percentage" field. This affects all subsequent Order deliveries.
Q: Can I add new payment gateways? Yes. Developers should create a new driver in backend/services/paymentDrivers/ and register it in the configuration seeder.
Q: How is multi-language handled? Language strings are managed through the i18n configuration. Admins can add translation files via the CMS language panel.
