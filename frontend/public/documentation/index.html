Global B2B Marketplace Documentation

Welcome to the official documentation for the **Global B2B Marketplace**. This platform is a full-stack, enterprise-grade solution designed to bridge the gap between global manufacturers (Suppliers) and bulk procurement professionals (Buyers).

Key Value Props:
  • Escrow-protected multi-currency transactions.
  • Built-in Messaging & Quotation engine for bulk negotiation.
  • AI-driven Product Discovery layer.
  • Multi-tenant Supplier Dashboards + Global Admin moderation.

Core Platform Features

The Global B2B Marketplace comes packaged with a comprehensive suite of features built specifically for bulk trade and international commerce:

  • Multi-Role System: Specialized portals and dashboards for Buyers, Suppliers, and Global Administrators.
  • Dynamic Catalog & Product Tiers: Multi-level category taxonomies with tiered pricing arrays, Minimum Order Quantity (MOQ) logic, and product customization queries.
  • RFQ & Negotiation Engine: Real-time messaging and Request for Quotation (RFQ) channels powered by Socket.io, allowing seamless custom price agreements.
  • End-to-End Visual Order Tracking: A visual milestone-based timeline tracking order status (Placed → Paid → Processing → Shipped → Delivered).
  • Multi-Gateway Payment Integration: Pre-integrated checkout flows supporting Stripe, PayPal, and Razorpay.
  • Robust Security & Verification: JWT-based session security, bot prevention using ReCAPTCHA v3, social logins (Facebook, LinkedIn), and built-in Admin KYC verification.
  • Global Localization: Support for multi-language display, dynamic currency conversions, and automated transactional email notification delivery via SMTP.
  • Premium Mobile-First Responsive Layout: A beautiful, responsive design with dynamic headers, search input container layout, and mobile-optimized product details image gallery layout.

Whether you are a developer tasked with scaling the infrastructure, or a client administrator managing day-to-day trades, this guide serves as your single source of truth.

1. Full Project Setup & Installation Guide

Prerequisites & Technology Stack

Before installing the platform on your local development machine or production server, ensure your environment meets the minimum architecture requirements.

  • Node.js: v18.x or v20.x LTS (Required for Frontend React app & Backend Express server).
  • Database: MongoDB v6.0 or higher.
  • Package Manager: NPM (comes with Node.js) or Yarn.
  • Web Server (Production): Nginx or Apache (for reverse proxy).
  • Process Manager: PM2 (for keeping the backend server alive in production).

Environment Variables (.env Configuration)

Both the frontend and backend require properly configured environments. Example environment variables:

# Backend (.env) PORT=5000 MONGODB_URI=mongodb://localhost:27017/alibaba_demo JWT_SECRET=your_super_secret_jwt_key STRIPE_SECRET_KEY=sk_test_... PAYPAL_CLIENT_ID=AcM... PAYPAL_APP_SECRET=EOf... FRONTEND_URL=http://localhost:3000 # Frontend (.env) REACT_APP_API_URL=http://localhost:5000/api REACT_APP_RECAPTCHA_SITE_KEY=your_recaptcha_v3_site_key

Important: Never commit your actual live `.env` files to Git version control. Always use `.env.example` as a template.

Backend Installation (Node.js/Express)

Follow these steps to initialize the core REST API backend.

cd backend/ npm install # To run local development server with hot-reload (Nodemon) npm run dev # To start production server npm start

Frontend Installation (React.js)

Start the user interface application which powers the Buyer, Supplier, and Admin portals.

cd frontend/ npm install npm start

The marketplace will automatically open at http://localhost:3000.

Database Seeding & Storage Setup

To populate your fresh MongoDB instance with default Admin users, predefined categories, and system settings, run the seeder script:

cd backend/ npm run seed

Storage Setup: Product images, invoices, and supplier documents are stored locally in the backend/uploads directory (or AWS S3/Cloudinary depending on your implemented driver in the `.env`). Make sure the uploads/ folder has public read permissions.

Queue & Cron Job Setup

Cron jobs are initialized within the Node runtime automatically upon starting the server. They handle background tasks such as:

  • Auto-canceling unpaid pending orders after 24 hours.
  • Compiling weekly analytics & sending summary emails.
  • Re-syncing subscription status for expired supplier plans.

Build & Deployment Steps (Production/Staging)

Deploying to a live Ubuntu/Linux server requires creating a React production build and using PM2 for the backend.

# 1. Build the Frontend cd frontend/ npm run build # (This outputs optimized static files to /build directory) # 2. Start Backend with PM2 cd backend/ npm install -g pm2 pm2 start server.js --name "alibaba-backend" pm2 save pm2 startup

You must then configure Nginx to serve the frontend/build folder on port 80/443, and proxy /api requests to localhost:5000.

Common Troubleshooting Steps

CORS Errors? Ensure your FRONTEND_URL in the backend `.env` precisely matches the domain the React app is served from (including http vs https).
Images not loading? Verify that your backend URL resolving static assets matches the explicit server IP/domain.

2. User Flows & System Architecture

End-to-End Transaction Lifecycle

For Clients (Business Logic)

The platform operates as an Escrow-driven marketplace. Buyers find products, place an order, and their payment is held securely. The Supplier is notified to ship the item. Only once the Buyer confirms they received the packages does the platform release the funds to the Supplier's virtual wallet, subtracting a marketplace commission.

For Developers (Technical Architecture)
  • Cart Splitting: The MongoDB Cart model splits items by supplier_id. Checkout generates one parent Transaction and multiple child Order documents.
  • Payment Escrow: Payments processed via Stripe/CCAvenue trigger a webhook that marks the Order as paid while holding financial distribution in a pending ledger.
  • Wallet DB Ledger: Upon the Buyer hitting "Confirm Receipt" (triggering /api/orders/:id/deliver route), the backend calculates the admin commission from Settings, and increments the Supplier's User.walletBalance.

Admin Workflow

For Clients (Business Logic)

Administrators are the overseers. Admins verify new suppliers to prevent fraud, moderate product listings, resolve buyer-supplier disputes, and manage website content (banners, global taxes, categories).

For Developers (Technical Architecture)
  • Role-Based Access Control (RBAC): Admins possess a JWT payload containing role: 'admin'. The backend Express isAdmin middleware protects all /api/admin/* HTTP routes.
  • Dynamic Configuration: Settings like global tax, categories, and hero sliders are not hardcoded. They are fetched dynamically from MongoDB collections (e.g., Settings) and mutated via the React dashboard.

Buyer Workflow

For Clients (Business Logic)

Buyers log in to source products across global verified suppliers. They can add ready-to-ship products to their cart, or for custom bulk orders, submit a Request for Quotation (RFQ) directly to a supplier to negotiate pricing.

For Developers (Technical Architecture)
  • RFQ Tracking System: Bypasses standard cart logic. Mongoose generates a Quotation document. Communication runs through a Socket.io channel until an agreed custom invoice is generated.
  • AI Search Layer: Leverages a background keyword heuristic route to map conversational Buyer queries (e.g., "cheap blue shoes") into exact Product schema $regex matches.

Supplier Workflow

For Clients (Business Logic)

Suppliers operate an independent storefront within the marketplace. They undergo KYC verification, upload catalogs with bulk-pricing tiers, fulfill orders with tracking numbers, and withdraw earnings natively via the dashboard.

For Developers (Technical Architecture)
  • Schema Ownership: Product models require an owner: ObjectId referencing the Supplier. A Supplier can only mutate products mapping strictly to their JWT ID.
  • Withdrawal System: Funds exist only as virtual integers in the DB until a Supplier submits a WithdrawalRequest. Financial APIs or manual Admin wires complete the transaction off-platform.

5. Supplier Panel Documentation

Store Dashboard Overview

Real-time shop performance analytics and pending action queues (Orders/RFQs).

Product & Inventory Management

Creating highly compliant listings ensures better ranking across the marketplace.

  • Basic Info: Title, precise categorization, brand, model.
  • Trade Details: Setting realistic Min. Order Quantities (MOQ), tiered pricing arrays (e.g. 1-100 units: $5, 100-500 units: $4).
  • Media: Main thumbnail and gallery additions.
  • Inventory tracking: SKU management and exact stock counts to prevent overselling.

Order Fulfillment & Shipping

The fulfillment pipeline mandates strict state updates:

  1. Pending: Waiting for buyer payment.
  2. Processing: Supplier is picking/packing.
  3. Shipped: Supplier attaches tracking ID and selects Shipping Carrier. Buyer notified.
  4. Delivered: Handover completed. Wallet funds released.

Earnings, Wallet & Payouts

For every successful order, minus platform commission, funds are assigned to the Supplier's Virtual Wallet.

The Withdrawals section allows Suppliers to submit a bank transfer request for a chosen amount. The Admin approves this manually off-platform and flags it as 'Processed'.

Quotation Responses

When Buyers submit tailored Requests for Quotation (RFQs) on custom setups or bulk inquiries, Suppliers receive them instantly.

  • Chat directly via the active real-time WebSockets tracking window.
  • Propose adjusted Minimum Order Quantities (MOQs) or revised tiered pricing structures.
  • Finalize quotes to immediately spawn a custom sub-order cart logic for the Buyer.

Business Profile Settings

A high-quality verified public profile directly correlates to B2B conversions. Settings encompass:

  • Company KYC: Upload Registration Certificates, ISO badges, and localized Tax ID info.
  • Capabilities Map: Define precise factory locations, main operational sectors, and global export nodes.
  • Media Richness: Embedded promotional Youtube links, banner galleries, and branded SEO tagging.

6. Frequently Asked Questions (FAQ)

Q: How do I change the marketplace commission?
A: Navigate to Admin > Settings > General Config. Update the "Commission Percentage" field. This affects all subsequent Order deliveries.
Q: Can I add new payment gateways?
A: Yes. Developers should create a new driver in backend/services/paymentDrivers/ and register it in the configuration seeder.
